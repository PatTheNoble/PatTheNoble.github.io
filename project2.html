Unit Manager

An efficient system for handling complex game AI in Unreal Engine 5

Technologies Used

  • C++
  • Unreal Engine 5

Problem: Unreal Engine’s AI and Movement Components Are Expensive

Unreal Engine’s default character and movement components are designed for high-quality player interactions, but they come with a high computational cost. In our case, the system could only handle about 10 characters simultaneously before suffering performance issues. This was a significant limitation, as our game wanted to support 100+ AI enemies interacting in real-time.

Solution: Custom Unit Manager (10x Performance Boost)

To address this, we developed a custom Unit Manager designed to efficiently handle a large number of AI entities. By optimizing how AI behavior is managed and offloading expensive movement calculations, we increased the system’s capacity by over 1000%, allowing the game to manage over 100 AI enemies at once with minimal impact on performance.

Multithreading and Tick Aggregation

Multithreading was a crucial part of the solution, allowing us to spread the computational load of AI decision-making across multiple CPU cores. We also implemented tick aggregation, which groups AI updates into batches, significantly reducing the overhead of individual entity updates. These optimizations further contributed to the performance gains.

Entity Component System (ECS) Architecture

To ensure scalability, we shifted to an Entity Component System (ECS) architecture. This allowed us to decouple AI behaviors from the Unreal Engine character framework, reducing the complexity and cost of managing AI entities. This shift allowed for more efficient data organization, especially by grouping similar components together in contiguous memory blocks. As a result, we achieved better cache coherence, meaning that data accessed by the CPU could be retrieved more quickly from memory, significantly reducing cache misses. This optimization led to faster AI updates and improved scalability, making it possible to handle a large number of entities with minimal performance impact.

Project Summary

  • 10x Performance Improvement
      • Enhanced the system to handle over 100 AI enemies simultaneously, compared to the original 10

  • Multithreading & Tick Aggregation
      • Distributed AI processing across multiple cores and aggregated updates to improve efficiency

  • Custom Avoidance System
      • Ensures smooth navigation and collision avoidance for large groups of AI

  • Entity Component System (ECS)
      • Decoupled AI logic from Unreal’s default character system for better scalability
